## Method

1. Read `docs/roadmap.md` end-to-end and enumerated the link graph (≈19 server-side docs + 5 webapp docs + reference docs not directly linked but reachable from those).
2. Captured production state on `media` as a single ground-truth snapshot:
   - `docker ps` / running containers
   - prod env vars (`docker exec journal-server env | grep ...`)
   - prod DB shape (`PRAGMA user_version`, table list, row counts, runtime_settings, pricing, users, jobs)
3. Dispatched **six parallel verification subagents**, each scoped to a doc cluster, all read-only. Each returned a structured findings report with severity, claim, reality, and suggested fix.
4. Synthesized findings, applied edits to the docs (some directly; api.md was delegated to a focused edit subagent because of its size).
5. Wrote this summary; committed and pushed.

Journal entries (`server/journal/*.md`, `webapp/journal/*.md`) were **not** audited — explicitly out of scope per the user's instructions.

## High-impact corrections (top of mind for future readers)

1. **`JOURNAL_API_TOKEN` is dead.** It was the legacy single-bearer-token from before the 2026-04-14 multi-user auth landing. The `api_bearer_token` field still exists in `config.py` but is not read by any production code path. Auth is now session cookies + per-user API keys. Required env var is **`JOURNAL_SECRET_KEY`** (used to sign sessions and reset tokens). The webapp's runtime-injection ceremony for `JOURNAL_API_TOKEN` is a committed no-op that just `echo`s a "not needed" line.
2. **OCR provider is Gemini in prod, not Anthropic.** Several docs (and the roadmap's Item 18) had been written assuming Anthropic OCR. Where the discussion is provider-specific (Anthropic 4096-token cache minimum), the docs now scope it explicitly. Where it's provider-agnostic, the docs now say so.
3. **Search uses hybrid pipeline; `mode=` is gone.** `api.md` had been describing the retired `mode=keyword|semantic` API. The webapp implements the current code's contract correctly; only the doc was wrong.
4. **`api.md` was missing ~30 endpoints** — all the auth, admin, settings/preferences, and several dashboard routes. Webapp consumers depended on the code, not the doc. Now documented.
5. **Two refactor "active" child plans were actually closed.** `refactor-repository-plan.md` and `refactor-item-6-exceptions-plan.md` had `Status: active` headers but every item had shipped weeks ago. Status flipped, with shipped-commit pointers.
6. **`mood-scoring.md` referenced a `--stale-only` CLI flag** in three places. The flag does not exist — stale-only is the default mode (you'd use `--force` to override). Replaced.
7. **`entity-tracking.md` Migration section** only documented `0004_entities.sql` despite referencing migrations through `0022` elsewhere in the same file. Rewritten with the full chronological timeline.
8. **Several refactor-era stale paths** (`mcp_server.py`, `api.py`, `cli.py`, `services/jobs.py`, `services/ingestion.py`, `entitystore/store.py`) were updated to point at the current packages.
9. **Stale corpus-size thresholds** in roadmap items 5 / 13 / 17 / D1 were updated from "30–50 entities" / "20 entries" / "5 entries" gating phrases to reflect the actual prod corpus (475 entities, 69 entries).
10. **Production compose drift** was discovered during the audit: the in-repo `compose.yml` pins `journal-server` to loopback, but `/srv/media/docker-compose.yml` exposes `0.0.0.0:8400`. Filed as `security-roadmap.md` Tier 2 item 2b.
