# Units 5, 6, 7 — integration test, test reach-in cleanup, .venv doc

Date: 2026-05-07

## Unit 5 — Chroma integration test in CI

`tests/integration/test_chroma_integration.py` — eight tests against a
real ChromaDB exercising every public method of `ChromaVectorStore`:

- `count()` (empty + populated)
- `add_entry()` round-trips count
- `search()` returns nearest chunk + respects `where=` filters + handles
  the empty-collection case
- `delete_entry()` removes one entry's chunks, leaves siblings
- `get_chunks_for_entry()` round-trips text, chunk_index, and embeddings
  (and returns empty for unknown entries)

### Marker registration

`pyproject.toml` now uses `--strict-markers`. `integration` is registered
so default `uv run pytest` runs skip the suite (matches the plan's
"register before adding `@pytest.mark.integration`" requirement).
Default unit run: 1794 passed, 8 deselected.

### CI

Two-job pipeline: existing `test` job runs `pytest -m "not integration"`,
new `integration-test` job uses `chromadb/chroma:latest` as a GitHub
Actions service container. Heartbeat polling happens on the runner
(curl available there) rather than relying on Docker's built-in
healthcheck — the chroma image historically lacked curl, and a 30-attempt
× 2-second loop on the runner is straightforward and self-contained.
`build-and-push` now needs both jobs green.

### Local quickstart

`docs/development.md` updated with a "Test markers" section. Locally,
the dev compose container exposes Chroma on `:8401`, so:

```bash
CHROMA_HOST=localhost CHROMA_PORT=8401 uv run pytest -m integration -v
```

## Unit 7 — `.venv` shebang foot-gun

`docs/development.md` gains a "Renaming the working directory" callout.
Symptom: after a `mv journal-server/ server/` (the layout the parent
`journal/` workspace uses), `uv run` works but the entry-point scripts
fail with a no-such-file error pointing at the *old* directory because
the existing `.venv/`'s shebangs are absolute. Fix: `rm -rf .venv && uv
sync` — uv only rewrites shebangs on a fresh build, not on an
incremental `uv sync`.

The doc-only change is the entirety of this unit; no code changed.
